# AquaBio-AgentRAG 重要文件与教学流程手册

这份文档用于教学：让学生按步骤理解 `F:\rag\AquaBio-AgentRAG` 项目每个重要文件是什么、负责哪一段流程、用了什么技术、应该先学哪里再学哪里。

---

## 0. 项目一句话说明

`AquaBio-AgentRAG` 是一个水下生物多模态 AgentRAG 项目。

它把：

```text
文本知识
图片 caption
PDF 图书
PDF 中抽取的真实图片
物种实体关系图谱
多轮会话记录
```

组织成一个可以问答的系统：

```text
用户问题 / 上传图片
  -> LangGraph 判断任务
  -> 调用 Chroma / RAG-Anything / MCP tools 检索证据
  -> Qwen 等大模型组织回答
  -> 前端显示答案、图片、证据、trace
```

## 13. 学生必须掌握的 5 个核心概念

### 13.1 Chunk

```text
把长文本切成小段，方便向量检索。
```

### 13.2 Embedding

```text
用 BGE-M3 把文本变成向量。
```

### 13.3 Vector Database

```text
Chroma 保存向量，负责相似度检索。
```

### 13.4 Graph / Relation

```text
图谱保存实体和关系。
例如：
Jasus lalandii --is_a--> Palinuridae
```

### 13.5 Agent Workflow

```text
LangGraph 让系统按步骤判断、检索、调用工具、生成回答、保存记忆。
```

## 14. 最后总结

这个项目可以按四句话教：

```text
1. 数据层：PDF、文本、图片、caption、实体关系。
2. 检索层：BGE-M3 + Chroma + PDF 图片索引 + 图谱。
3. 工具层：Chroma MCP Server + RAG-Anything MCP Server。
4. Agent 层：LangGraph 根据问题自动选择工具，调用检索结果，让大模型回答并保存记忆。
```

最关键的学习方法：

```text
不要只看最终回答。
一定要看 --json 输出里的 route、selected_tools、trace、tool_calls、evidence。
这些才是真正证明系统流程跑通的证据。
```
