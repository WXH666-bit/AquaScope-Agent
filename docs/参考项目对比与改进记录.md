- `m13_streamlit/s04_agent_multi_session.py`
- `m11_mcp_advanced/mcp_bridge.py`
- `m11_mcp_advanced/s01_agent_stream.py`

可学习点：

| 设计 | AquaBio 的采用方式 |
| --- | --- |
| 驾驶舱展示当前 Agent、工具和会话统计 | 增加 Agent 驾驶舱、ReAct 阶段和 MCP 活动 |
| SQLite 持久会话 | 已有 `web_app.sqlite` 与 LangGraph checkpoint |

没有直接照搬：

- 未引入 Agents SDK swarm，因为当前单领域受控 Agent 更易审计。
- 未把任意 MCP 工具全部交给模型，避免免费/限流模型误调用高成本工具。

## 2. all-in-rag-main

重点参考：

- `code/C9/rag_modules/intelligent_query_router.py`
- `code/C9/rag_modules/hybrid_retrieval.py`
- `code/C9/rag_modules/generation_integration.py`
- `code/C4/07_rerank_and_refine.py`
- `docs/chapter4/11_hybrid_search.md`
- `docs/chapter4/14_query_rewriting.md`

可学习点：

| 设计 | AquaBio 的采用方式 |
| --- | --- |
| 查询复杂度与数据源路由 | LangGraph router + source_selection |
| BM25、向量、图检索融合 | Chroma + 本地 BM25 + LightRAG，加权 RRF |
| 查询重写与多数据源分发 | query_rewrite + 物种追问解析 + Web research plan |
| 重排与上下文压缩 | 领域权重重排、去重、最多 24 条上下文 |
| 生成质量反馈 | 新增每轮回答反馈表与 API |
| 评估闭环 | answer → guard → evaluate → retry |

没有直接照搬：

- 不引入 Milvus：当前 772 条主向量和 PDF 图片索引使用 Chroma 足够。
- 不引入 Neo4j：当前图谱规模使用 NetworkX + LightRAG 本地存储更轻量。
- 不引入独立 ColBERT：会额外占用模型磁盘和内存；先保留 BGE-M3 重排。
