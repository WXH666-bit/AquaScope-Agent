## 3. 真实 Agent 与 MCP

验证问题：

```text
请调用PDF知识图谱简要说明海星的外观特征，并标注证据。
```

关键轨迹：

```text
react_tool_plan:
step=1;mode=native_function_calling;tools=pdf_retriever

retrieval:
text=0,web=0,image=0,pair=0,pdf=9,
mcp_graph=9,mcp_pdf_images=0,
filter_species=starfish,elapsed=0.93s

answer_generation:
qwen:qwen3.7-plus:19.20s
```

Qwen 使用原生 function calling 选择工具，`pdf_retriever` 经 MCP 调用
RAG-Anything/LightRAG，9 条最终证据来自 `lightrag_graph`。总请求约
24.5 秒，无降级警告。完整响应位于：

```text
data/mrag/logs/final_react_verification.json
```

## 7. Windows MCP 退出清理

Chroma MCP 使用 stdio 子进程。客户端现在会在 `ClientSession` 和 stdio
transport 退出后，为 Windows Proactor 管道关闭回调保留一个短事件循环
清理周期，再结束同步调用。

重复列出 Chroma 8 个工具和 RAG-Anything 6 个工具后进程正常退出，未再出现
`Event loop is closed` 或 `unclosed transport` 警告。
