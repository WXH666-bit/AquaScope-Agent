# AquaBio 本地前后端聊天助手

## 1. 已实现架构

```text
Streamlit 前端（8510）
  -> FastAPI 后端（8000）
  -> ChatService
  -> LangGraph AquaBioMRAGWorkflow
  -> Chroma MCP / RAG-Anything MCP
  -> Chroma / LightRAG / NetworkX / BGE-M3
  -> SQLite + ConversationStore + 本地文件系统
```

前端不直接调用 LangGraph。所有聊天、上传、会话和证据数据都通过 FastAPI
接口完成，因此后续可以在不修改后端的情况下替换成 React。

## 2. 主要功能

```text
文本问答
图片问答
图文联合问答
上传 PDF 并将文本作为本轮 E0 证据
PDF 实体样例图片检索与展示
多轮会话记忆
历史会话、搜索、收藏、删除
会话 JSON 导出
证据卡片、页码、图片和关系路径
LangGraph 执行轨迹
MCP Server 与工具列表
功能开关真实控制 LangGraph
SQLite 保存消息、附件、证据和轨迹
```

## 3. 一键启动

在 PowerShell 中执行：

```powershell
cd F:\rag\AquaBio-AgentRAG
.\start_chat_assistant.cmd
```

访问：

```text
聊天界面：http://127.0.0.1:8510
FastAPI 文档：http://127.0.0.1:8000/docs
健康检查：http://127.0.0.1:8000/api/health
```

停止：

```powershell
.\stop_chat_assistant.cmd
```

也可以显式绕过当前进程的 PowerShell 执行策略：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start_chat_assistant.ps1
```

## 6. 功能开关

聊天请求中的开关会进入 `AquaBioState.runtime_options`：

```text
memory_enabled
rag_enabled
vision_enabled
pdf_enabled
mcp_enabled
image_search_enabled
citation_enabled
log_enabled
```

行为：

```text
memory_enabled=false
  -> 不读取历史记忆，也不写入 ConversationStore

vision_enabled=false
  -> 上传图片时跳过 VLM

pdf_enabled=false
  -> 不调用 PDF Retriever，不读取本轮 PDF

mcp_enabled=false
  -> LightRAG MCP 跳过，PDF 图片查询使用本地 Python fallback

log_enabled=false
  -> 不写入 Web SQLite 消息、证据和轨迹
```

## 7. 保存位置

```text

  -> 前端会话、消息、附件、证据、轨迹

  -> LangGraph 会话语义记忆

data/mrag/sessions/langgraph.sqlite
  -> LangGraph Checkpointer

data/mrag/uploads/image/
  -> 用户上传图片

data/mrag/uploads/pdf/

data/mrag/sessions/exports/
  -> 会话导出 JSON
```

## 9. 后台窗口与长任务状态

`start_chat_assistant.cmd` 默认打开两个可见命令窗口：

```text
AquaBio FastAPI Backend
  -> 显示 HTTP 请求、模型加载、MCP 调用
  -> 每轮显示 [CHAT START]、[CHAT DONE] 或 [CHAT ERROR]

AquaBio Streamlit Frontend
  -> 显示 Streamlit 启动和页面异常
```

聊天页面不会再同步盲等。前端先创建后台任务，然后每秒查询任务状态：

```text
排队等待
初始化 Agent
执行 LangGraph
完成 / 执行失败
```

页面会显示任务 ID、当前阶段和已耗时。超过 240 秒会停止前端等待并提示
检查 FastAPI 后台窗口。

LightRAG Hybrid MCP 单次最多等待 20 秒。发生超时后会触发 5 分钟熔断，
后续问题直接降级到 Chroma + BM25，避免每个问题反复长时间等待。回答的
警告面板会说明是否发生了降级。

重复执行启动命令时，如果服务已经正常运行，脚本只会显示现有 PID 和地址，
不会再次创建端口冲突进程。
