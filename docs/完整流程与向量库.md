# AquaBio-MRAG 严格流程与向量库

## 1. 当前实现

本项目当前主流程位于 `src/aquabio_mrag/`，重点是多模态 RAG、持久化向量库、LangGraph Agent 和 MCP 工具，不把目标检测作为当前必需能力。

已落地的数据规模：

| 数据类型 | 数量 |
|---|---:|
| 水下生物类别 | 20 |
| 物种卡片 | 20 |
| 长文本知识块 | 160 |
| 图片及 caption | 200（每类 10 张） |
| 图片-文本配对 | 200 |
| PDF 文本块 | 192 |
| Chroma 向量总数 | 772 |

20 类包括：海星、海胆、海参、扇贝、水母、章鱼、鱿鱼、虾、蟹、龙虾、小丑鱼、海马、黄貂鱼、鲨鱼、海龟、珊瑚、海葵、蝠鲼、鳗鱼、海豚。

`.env` 的关键配置：

```env

OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
OPENROUTER_MODEL=nex-agi/nex-n2-pro:free

MRAG_EMBEDDING_MODEL=BAAI/bge-m3
```

`MRAG_MODEL_CACHE` 用于避免 BGE-M3 占用空间较小的 C 盘。路径可按机器实际磁盘修改。

向量库构建和离线检索不需要 OpenRouter key。已贴到聊天或提交到公开文件的 key 应立即吊销，不要继续使用。


## 7. 检索与重排

检索入口位于 `src/aquabio_mrag/retrieval.py`。按任务选择来源：

| 任务 | 检索来源 |
|---|---|
| 纯文本问答 | 物种卡、长文本、PDF |
| 纯图片问答 | 图片 caption、图文配对、物种文本、PDF |
| 图文问答 | 查询文本 + VLM caption，检索全部相关来源 |
| 对比问答 | 重点提高视觉特征、识别提示和多物种文本权重 |
| 来源追溯 | 优先返回来源页、PDF 标题和页码 |
| PDF 问答 | 只优先检索 `pdf_chunk` |

重排公式：

```text
final_score =
  0.45 * semantic_similarity
  + 0.20 * species_match
  + 0.15 * source_weight
  + 0.10 * chunk_weight
  + 0.10 * keyword_overlap
```

## 8. LangGraph 严格工作流

`src/aquabio_mrag/workflow.py` 使用显式状态和节点：

1. `router`：识别六种任务类型。
2. `rewrite`：按任务补充检索意图。
3. `source_selection`：选择 VLM、文本、图片、配对和 PDF 检索器。
4. `vision`：图片输入通过 OpenRouter 生成 caption、视觉特征和候选类别。
5. `retrieval`：执行多源向量检索与重排。
6. `context`：去重，生成 `[E1]` 形式证据块。
7. `answer`：只根据 caption 和检索证据回答。
8. `evaluation`：检查上下文、引用、图片一致性。
9. `retry`：最多按配置重试 2 次。
10. `finalize`：输出答案和完整 trace。

状态模型和文档模型使用 Pydantic `extra="forbid"`，额外字段会被拒绝，避免节点间随意传递不受控数据。

## 10. MCP 工具

启动 stdio MCP 服务器：

```cmd
.\.venv\Scripts\python.exe -m aquabio_mrag.mcp_server
```

实际注册的工具：

| 工具 | 用途 |
|---|---|
| `search_species_text` | 检索物种卡和长文本 |
| `search_image_captions` | 检索图片 caption |
| `search_multimodal` | 联合检索图文配对、文本和 PDF |
| `search_pdf` | 检索 PDF 并返回页码 |
| `generate_image_caption` | 调用 OpenRouter 分析本地图片 |
| `get_source_detail` | 按文档 ID 获取原始内容与元数据 |

LangGraph 负责状态、条件边、重试和最终回答；MCP 负责把可复用能力暴露为严格工具。两者职责分开。

## 12. 关键限制

1. 当前“图片识别”来自 VLM caption，不是有边界框和 mAP 指标的目标检测。
2. 当前不使用 CLIP；图片通过 caption 和图文配对进入 BGE-M3 文本向量空间。
3. iNaturalist 中部分图片为 `CC-BY-NC` 或 `CC-BY-NC-SA`，只适合遵守许可的非商业研究和教学。
4. 免费 OpenRouter 模型可能拥堵、调整能力或下线，因此模型名必须通过环境变量管理。
5. PDF 图像抽取当前为可选项，本次五份 PDF 生成了 192 个文本块，没有生成独立 `pdf_figure`。
